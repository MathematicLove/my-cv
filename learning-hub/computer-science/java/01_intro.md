# Разница JDK, JRE, JVM

JDK - предоставляет инструменты и библиотеки для разработки Java приложений, работающих с JRE и JVM. JRE (Java Runtime Environment) предлагает библиотеки и JVM, необходимые для запуска Java-программ. JVM (Java Virtual Machine) выполняет скомпилированный байт-код Java в системе.

> *Важно отметить, что хоть концепция Java - Write Once Run Anywhere, действует, но JVM является платформо-зависимой для каждой ОС.*

JDK - это программное обеспечение, которое позволяет реализовывать программы на Java. JDK включает в себя JRE и следующие инструменты:
- компилятор `javac`, отладчик `debugger` и др. инструменты по типу `jar`, `javadoc`;
Нужен что бы, позволять разработчику - писать, компилировать, отлаживать код.

**Компоненты JDK**:
- JRE (JVM + библиотеки);
- Инструменты разработчика:
	- отладчик
	- `jar`
	- `javadoc`

> **JDK - не позволяет запускать программу!, он нужен что бы ее разрабатывать;**
> **JDK - платформо-зависимый.**

### Как работает JRE
1. Загрузка классов - загружает файлы `.class`, в память;
2. Верификация байт-кода - обеспечение безопасности;
3. Исполнение - использует JVM (интерпретатор + JIT компилятор) что бы считывать инструкции и вызывать системные методы.

### Как работает JVM

> Байт-код может исполняться только в JVM
   JVM - платформо-зависимый;
   JVM - использует JIT компилятор для скорости выполнения;

1. Загрузка - ClassLoader загружает байт-код в память;
2. Линковка - Верификация, подготовка, разрешение;
3. Инициализация - Инициализирует конструкторы классов и статические блоки/классы/методы;
4. Выполнение - Интерпретация / компиляция байт-кода в машинный код.
